
# 🧪 Projeto 3 – Checklist e Testes de Layout

Este repositório contém as atividades desenvolvidas na **3ª sprint**, com foco na criação de **checklists**, execução de **testes de layout** e validação de **funcionalidades críticas** da aplicação.

O projeto tem como objetivo garantir a consistência visual, a usabilidade da interface e o correto funcionamento de recursos essenciais, especialmente relacionados ao fluxo de **reserva**.

---

## 📌 Objetivo

Assegurar a qualidade da aplicação por meio de:

- Validação de layout em diferentes resoluções e navegadores  
- Execução de checklists estruturados  
- Testes funcionais de componentes críticos  
- Verificação da lógica de funcionalidades relacionadas à reserva  

---

## 🖥️ Ambientes de Teste

Os testes foram realizados considerando diferentes cenários de uso:

- **Google Chrome – 800x600** (foco principal)  
- **Mozilla Firefox – 1920x1080**  

Essa abordagem permite identificar problemas de responsividade e inconsistências entre navegadores.

---

## ✅ Escopo dos Testes

### 🎯 Testes de Layout

- Verificação de alinhamento, espaçamento e responsividade  
- Validação da consistência visual entre diferentes resoluções  
- Identificação de quebras de layout e elementos desalinhados  

---

### 📋 Checklists

Foram elaborados checklists para garantir cobertura sistemática dos testes, incluindo:

- Janela **"Método de pagamento"**  
- Janela **"Adicionar cartão"**  
- Elementos visuais e interativos da interface  

---

### 🔘 Testes Funcionais

#### Botão "Reservar"

- Validação da lógica de funcionamento  
- Comportamento em cenários válidos e inválidos  
- Respostas do sistema às interações do usuário  

#### Recursos de Reserva

- Testes dos fluxos relacionados à reserva  
- Validação das regras de negócio  
- Verificação da integridade das ações realizadas
